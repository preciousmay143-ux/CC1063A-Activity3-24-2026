Laboratory 1: Tools & Architecture Setup
Project Purpose this repository is built for Laboratory 1: Tools & Architecture Setup and the goal is to First set up essential development tools likeVS Code as the code editor and Git for version control. Second create a well-organized structure that keeps code easy to find, update, and scale and the Last is to define clear boundaries for different parts of the project to avoid confusion as work progresses.
And our chosen folder structure are 1.src/ - it holds all working code for the project this is where we build our solution. 2.src/components - its stores reusable user interface/UI parts for example like buttons, forms, cards and these only handle how things look and how users interact with them no behind the scenes work here. 3.src/logic/ - it manages core project rules and workflows for example calculating values and validating inputs. This layer connects what the user sees to how data is handled. 4.src/data/ - deals with all data-related tasks like reading from files, connecting to APIs, storing information. It hides complex data work from other parts of the project. 5.docs/ - it contains extra guides, diagrams, or notes that help explain the project like how to set up the environment and design decisions. 6.tests/ - has files to check if our code works correctly like testing if a button triggers the right action and if calculations are accurate. We chose this structure because it groups similar work together so anyone working on the project can immediately find where to add or change code and it also prevents one part of the project from interfering with another.
